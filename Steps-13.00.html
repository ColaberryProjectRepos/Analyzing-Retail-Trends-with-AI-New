Creating Dimension Tables 13

Fact tables store quantitative data that is usually generated by business processes or events. These tables capture measurable metrics, which are typically numeric values used for analysis and reporting.
Dimension tables provide context to the data in fact tables by storing descriptive information. They typically contain attributes that describe the "who," "what," "where," "when," and "why" of the facts.

After identifying the fact and dimension tables, I designed and implemented these tables in the data warehouse. This process also included developing and executing ETL (Extract, Transform, Load) processes using SQL queries to ensure the accurate and efficient population of the tables with relevant data.

Note: Microsoft Fabric does not support primary keys, so use ChatGPT to obtain code for populating fact and dimension tables to address this issue.
